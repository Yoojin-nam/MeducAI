# 테이블 열 넓이 및 줄바꿈 규칙 분석

## 현재 구현 상태

### 1. Entity Name 컬럼 (첫 번째 컬럼)

#### 줄바꿈 규칙
- **"&" 뒤에서 줄바꿈**: `"질환명 & 영어명"` → `"질환명 &<br/> 영어명"`
- **"(" 앞에서 줄바꿈**: `"한글명 (영어명)"` → `"한글명 <br/>(영어명)"`
- `splitLongWords=True`: 공백, &, (에서 자연스러운 줄바꿈 허용

#### 최소 너비 계산
- **세그먼트별 측정**: "&"와 "("로 분할된 각 세그먼트의 너비 측정
- **전체 텍스트 측정**: 한 줄에 들어갈 경우를 대비해 전체 텍스트 너비도 측정
- **가장 긴 단어 추적**: 오버플로우 방지를 위해 가장 긴 단어 너비 추적
- **안전 마진**: 20% (다른 컬럼은 30%)
- **표준 마진**: 0.5cm (다른 컬럼은 1.0cm)
- **패딩**: 좌우 각 3pt

#### Content Score 가중치
- **40% 감소**: 첫 2개 컬럼의 Content score를 0.6배로 조정
- **목적**: 남은 공간 분배에서 불리하게 처리하여 나머지 4개 컬럼과 균형 유지

#### 최소 너비 보장
- **우선순위**: 최우선 (다른 컬럼보다 먼저 처리)
- **공간 가져오기**: 다른 컬럼에서 최대 80%까지 축소 가능 (이전 95%에서 완화)

### 2. 질환/개념 컬럼 (두 번째 컬럼)

#### 줄바꿈 규칙
- **세미콜론("; ") 뒤에서 줄바꿈**: `"항목1; 항목2"` → `"항목1;<br/> 항목2"`
- **쉼표(", ")에서 자연스러운 줄바꿈**: ReportLab이 공백에서 자동 줄바꿈
- `splitLongWords=True`: 긴 단어도 필요시 줄바꿈 가능

#### 최소 너비 계산
- **세그먼트별 측정**: 쉼표로 분할된 각 세그먼트의 너비 측정
- **전체 텍스트 측정**: 한 줄에 들어갈 경우를 대비해 전체 텍스트 너비도 측정
- **가장 긴 단어 추적**: 3자 이상 단어만 추적
- **안전 마진**: 25% (Entity Name 20%, 다른 컬럼 30%의 중간값)
- **표준 마진**: 0.7cm (Entity Name 0.5cm, 다른 컬럼 1.0cm의 중간값)
- **패딩**: 좌우 각 3pt

#### Content Score 가중치
- **45% 감소**: 두 번째 컬럼의 Content score를 0.55배로 조정 (Entity Name은 0.5배)

#### 최소 너비 보장
- **우선순위**: 두 번째 우선순위 (Entity Name 다음)
- **공간 가져오기**: 다른 컬럼에서 최대 85%까지 축소 가능

### 3. 나머지 4개 컬럼 (3-6번째)

#### 줄바꿈 규칙
- **세미콜론("; ") 뒤에서 줄바꿈**: `"항목1; 항목2"` → `"항목1;<br/> 항목2"`
- **쉼표(", ")에서 자연스러운 줄바꿈**: ReportLab이 공백에서 자동 줄바꿈
- `splitLongWords=True`: 긴 단어도 필요시 줄바꿈 가능

#### 최소 너비 계산
- **세그먼트별 측정**: 쉼표로 분할된 각 세그먼트의 너비 측정
- **전체 텍스트 측정**: 한 줄에 들어갈 경우를 대비해 전체 텍스트 너비도 측정
- **가장 긴 단어 추적**: 3자 이상 단어만 추적
- **안전 마진**: 30%
- **표준 마진**: 1.0cm
- **패딩**: 좌우 각 3pt

#### Content Score 가중치
- **정상 계산**: 헤더 길이 30% + 평균 데이터 길이 70%

## 열 너비 할당 전략

### 1단계: 최소 너비 계산
- 각 컬럼의 최소 너비를 내용 기반으로 계산
- Entity Name: 세그먼트별 측정 (줄바꿈 고려)
- 다른 컬럼: 쉼표로 분할된 세그먼트별 측정

### 2단계: 공간 부족 시 비례 축소
- 모든 컬럼의 최소 너비 합계가 사용 가능 너비를 초과하면
- 모든 컬럼을 비례적으로 축소: `scale_factor = available_width / total_min_width`

### 3단계: 남은 공간 분배
- 최소 너비 합계가 사용 가능 너비 이하이면
- 모든 컬럼에 최소 너비 할당
- 남은 공간을 Content Score에 비례하여 분배
  - 첫 2개 컬럼: Content Score 40% 감소
  - 나머지 4개 컬럼: 정상 Content Score

### 4단계: 최소 너비 보장
- 모든 컬럼이 최소 너비를 만족하도록 보정
- Entity Name 컬럼이 최우선
- 부족한 공간은 다른 컬럼에서 가져옴 (최대 80% 축소)

## 발견된 문제점 및 개선점

### ✅ 해결된 문제
1. **Entity Name에서 "&" 줄바꿈**: "&" 뒤에서 줄바꿈하도록 수정 완료
2. **첫 2개 컬럼 과도한 너비**: 
   - Entity Name: Content Score 50% 감소, 마진 0.5cm로 감소, 안전 마진 20%로 감소
   - 두 번째 컬럼: Content Score 45% 감소, 마진 0.7cm로 감소, 안전 마진 25%로 감소
   - 우선순위 조정: 첫 2개 컬럼 모두 우선순위 상향, 공간 가져오기 제한 완화
3. **Content Score 가중치 파라미터화**: ✅ 완료
   - CLI 파라미터 `--content_score_weight_col0`, `--content_score_weight_col1` 추가
   - 기본값: Entity Name 0.5, 두 번째 컬럼 0.55
   - 실험적 조정 가능
4. **세미콜론 줄바꿈 개선**: ✅ 완료
   - 세미콜론 뒤에 공백이 없어도 줄바꿈 추가
   - HTML 태그 시작 전에는 줄바꿈 추가하지 않음 (이미 처리됨)

### 🔍 추가 개선 가능한 점

#### 1. Content Score 계산 방식 ✅ (완료)
- **완료**: 컬럼별 가중치를 CLI 파라미터로 관리 가능하도록 개선

#### 4. 최소 너비 보장 로직 ✅ (완료)
- **현재**: Entity Name이 최우선, 두 번째 컬럼이 두 번째 우선순위
- **완료**: 두 번째 컬럼도 우선순위 조정 완료 (Entity Name 다음)
- **완료**: 두 번째 컬럼도 공간 가져오기 제한 완화 (최대 85% 축소 가능)

#### 5. 폰트 크기 및 측정 정확도
- **현재**: 8pt 폰트, `stringWidth`로 측정 후 20% 마진 추가
- **문제**: 한글/영문 혼용 텍스트에서 측정 오차 가능
- **개선안**: 실제 렌더링 테스트를 통한 마진 조정

#### 6. 동적 조정 파라미터
- **현재**: 마진, Content Score 배율 등이 하드코딩됨
- **개선안**: 설정 파일이나 CLI 파라미터로 관리하여 실험적 조정 가능
