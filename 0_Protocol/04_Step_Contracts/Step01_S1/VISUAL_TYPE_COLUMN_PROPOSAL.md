# Visual Type Category별 Column 제목 제안

**생성일:** 2025-12-19  
**수정일:** 2025-12-23 (v11)  
**목적:** 각 `visual_type_category`에 맞는 최적화된 master table column 제목 제안  
**상태:** 제안 단계 (사용자 검증 필요)

**변경 사항 (v11):**
- `Comparison`, `Algorithm`, `Classification`, `Sign_Collection` 카테고리 제거 (연구에서 사용되지 않음)
- 해당 카테고리 섹션은 주석 처리됨

---

## 현재 상태

**현재 (모든 카테고리 공통):**
```
| 범주 | 항목(질환/개념) | 영상 소견 키워드 | 모달리티별 핵심 소견 | 병리·기전/특징 | 감별·함정(전문의 시험 포인트) |
```

**문제점:**
- 일부 카테고리(예: Equipment, QC)에는 "질환/개념", "병리·기전" 같은 용어가 적합하지 않을 수 있음
- **Note (v11):** Algorithm, Comparison, Classification, Sign_Collection categories have been removed as they were not used in the study.
- 각 카테고리의 특성에 맞는 더 정확한 column 제목이 필요할 수 있음

## 수정 요청사항 (사용자 결정)

1. **첫 번째 컬럼:** "범주" → **"Entity name"**으로 변경
2. **마지막 컬럼:** 항상 **"시험포인트"**로 통일

## 제안안: Category별 Customized Columns

### A) Anatomy_Map

**제안 column:**
```
| Entity name | 해부학적 구조 | 위치/경계 | 혈관/신경 관계 | 정상변이 | 수술/접근 경로 | 시험포인트 |
```

**이유:**
- 첫 번째 컬럼: "범주" → "Entity name" (실제 항목명)
- "질환/개념" → "해부학적 구조" (더 정확)
- "병리·기전" → "혈관/신경 관계" (해부학적 관계 강조)
- 마지막 컬럼: 항상 "시험포인트"로 통일

## 구현 고려사항

### 옵션 1: 프롬프트에 조건부 로직 추가
- 프롬프트에서 `visual_type_category`에 따라 다른 column 제목 사용
- 장점: 유연성
- 단점: 프롬프트 복잡도 증가, 검증 어려움

### 옵션 2: 카테고리별 프롬프트 분리
- 각 카테고리별로 별도 프롬프트 파일 생성
- 장점: 명확성, 유지보수 용이
- 단점: 파일 수 증가

### 옵션 3: 현재 구조 유지 + 범주 토큰으로 보완
- Column 제목은 공통 유지
- "범주" column의 토큰으로 카테고리 특성 반영
- 장점: 단순성 유지
- 단점: 일부 카테고리에서 부적합할 수 있음
