# 비열등성 검정의 목적 및 표본 크기 근거 (Rationale Discussion)

**Status:** Discussion Document  
**Date:** 2025-12-23  
**Context:** 비열등성 검정이 "필수"인지 "최적화 도구"인지에 대한 논의

---

## 1. 핵심 질문

**질문:** 비열등성을 점검해서 저렴한 모델을 쓰고 싶었지만, 열등하다고 나온다해도 문제가 없어. 비싼 모델 돌리면 되거든? 그럼 지금 Delta=0.5에 18 세트인게 무리가 아닐까? 비열등을 꼭 점검하려는게 아니면? 그런의도로도 비열등성 점검을 하니?

---

## 2. 현재 설계의 목적 (Canonical Documents)

### 2.1 비열등성 검정의 공식 목적

**QA_Framework.md Section 2.1:**
- Step S0는 6-arm 실험을 통해 **단일 Deployment Model**을 선정하고, 이후 전체 생성에서 **설정 변경 금지**가 가능한 근거(Methods/Discussion)를 확보한다.

**Model_Selection_Rationale.md Section 5.2:**
- **Quality:** Overall Card Quality를 non-inferiority endpoint로 사용
- **Statistical logic:** Non-inferiority (Reference: Arm E, Δ = 0.5 on 1–5 Likert scale)
- 이를 통해 **저비용 arm 채택 가능성**을 통계적으로 방어 가능하게 설계하였다.

### 2.2 Fallback Policy

**QA_Framework.md Section 2.10 (Decision Rules):**

```
4. Fallback Policy:
   - 비열등 arm이 없으면: Baseline Arm (A) 또는 Arm E (High-End) 또는 Arm F로 전체 생성 진행
```

**해석:** 비열등성 검정이 실패해도 High-End 모델을 사용할 수 있으므로, 비열등성 검정은 **"필수"가 아니라 "최적화 도구"**로 작동한다.

## 3. 논리적 분석

### 3.1 비열등성 검정이 "필수"인 경우

만약 비열등성 검정이 **필수 검증 절차**라면:
- 저비용 모델을 채택하려면 비열등성을 입증해야 함
- 비열등성 검정 실패 시 대안 선택지가 제한적
- 따라서 **엄격한 통계적 기준(n=18, Δ=0.5, 95% CI)**이 타당

**문제점:** Fallback policy가 존재하므로, 비열등성 검정이 "필수"는 아님

### 3.2 비열등성 검정이 "최적화 도구"인 경우

만약 비열등성 검정이 **cost optimization 도구**라면:
- "가능하면 저렴한 모델을 쓰고 싶다"는 목적
- 비열등성 검정 실패 시 High-End 모델 사용 (문제 없음)
- 따라서 **더 관대한 기준(n<18, Δ>0.5, 90% CI)**도 고려 가능

**현재 설계:** n=18, Δ=0.5, 95% CI는 "필수 검증" 수준의 엄격함

---

## 4. 표본 크기 및 Margin 선택의 함의

### 4.1 현재 설계 (n=18, Δ=0.5)

**근거 (S0_Non-Inferiority_QA_and_Final_Model_Selection_Protocol.md Section 4):**
- n=18은 **non-inferiority early-phase pilot으로서 실무적 하한선**으로 정의
- Δ = 0.5는 1–5 Likert scale에서 **half of one step** (교육적으로 의미 있는 최소 허용 저하)

**의도:**
- 논문에서 비열등성을 **통계적으로 방어 가능하게** 주장하기 위함
- "저비용 모델이 고비용 모델 대비 비열등하다"는 결론을 논문에 포함 가능

### 4.2 만약 비열등성이 "최적화 도구"라면

**더 관대한 설계 옵션:**
- **n < 18:** 예) n=12, n=10 (더 작은 샘플)
- **Δ > 0.5:** 예) Δ=0.6, Δ=0.7 (더 관대한 margin)
- **90% CI:** 95% 대신 90% (one-sided α=0.05)

**장점:**
- 평가 워크로드 감소
- 실행 속도 향상
- 비용 절감

**단점:**
- 논문에서 비열등성을 "통계적으로 방어 가능하게" 주장하기 어려움
- Reviewer가 "표본이 너무 작다"고 지적할 가능성

---

## 5. 실무적 고려사항

### 5.1 논문 출간 목적

**만약 논문에서 비열등성을 주장하려면:**
- 현재 설계 (n=18, Δ=0.5, 95% CI)가 더 타당
- Reviewer가 "표본 크기가 충분한가?", "Margin이 적절한가?" 질문 가능
- Power analysis나 margin justification 필요

**만약 논문에서 비열등성을 주장하지 않는다면:**
- "여러 arm을 비교하여 cost-efficient option을 선택했다"고 기술
- 비열등성 검정은 "보조 정보"로만 보고
- 더 작은 n, 더 큰 Δ도 가능

### 5.2 실제 배포 의사결정

**Fallback policy가 존재하므로:**
- 비열등성 검정 실패 → High-End 모델 사용 (문제 없음)
- 비열등성 검정은 **"가능하면 저렴한 걸 쓰고 싶다"는 최적화 목적**
- 따라서 통계적 엄격함보다는 **실무적 실행 가능성**이 더 중요할 수 있음

## 6. 권장 사항

### 6.1 목적 명확화

**질문:** 비열등성 검정의 목적이 무엇인가?

1. **Option A: 논문에서 비열등성을 주장 (Primary Purpose)**
   - 현재 설계 유지 (n=18, Δ=0.5, 95% CI)
   - Power analysis 및 margin justification 문서화

2. **Option B: Cost optimization을 위한 탐색적 비교 (Secondary Purpose)**
   - 더 관대한 설계 고려 (n=12-15, Δ=0.6-0.7, 90% CI)
   - 논문에서는 "비열등성 검정"보다 "비용 효율성 비교"로 기술

### 6.2 문서 업데이트 필요사항

**QA_Framework.md 또는 관련 문서에 다음 내용 추가 권장:**

1. **비열등성 검정의 목적 명시:**
   - Primary: 논문에서 비열등성을 주장하기 위함
   - Secondary: Cost optimization을 위한 실무적 의사결정 지원

2. **Fallback policy와의 관계 명확화:**
   - 비열등성 검정 실패 시에도 High-End 모델 사용 가능
   - 따라서 비열등성 검정은 "필수"가 아니라 "최적화 도구"

3. **표본 크기 및 Margin 선택 근거:**
   - n=18, Δ=0.5가 논문 출간 목적에 적합함을 명시
   - 만약 논문 목적이 아니라면 더 관대한 기준도 가능
